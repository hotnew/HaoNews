# 2026-03-18 HD 身份升级计划

## 目标

在不破坏现有单密钥身份模型的前提下，为 Hao.News 好牛Ai 增加分层确定性身份支持。

## 当时的实际状态

- 仓库当时只支持独立 Ed25519 身份文件
- `internal/haonews/identity.go` 没有 HD 字段、助记词处理或派生逻辑
- `cmd/haonews/main.go` 只支持 `identity init`
- 消息签名校验只验证消息中已经写入的具体公钥
- `writer policy` 还没有 HD 父子信任模型

## 约束

- CLI 输出、日志和测试中都不能打印助记词、seed 或私钥
- 现有独立身份文件必须保持兼容
- 相同助记词 + 相同派生路径的结果必须确定一致
- 优先按 P0 / P1 顺序推进，后续再扩展策略和 API

## 阶段计划

### Phase 1: P0 基础层

状态：已完成

- 增加 Ed25519 的 HD 派生基础能力
- 增加助记词生成与恢复
- 扩展 `AgentIdentity` 以表示 HD 根身份和派生身份
- 增加 CLI：
  - `identity create-hd`
  - `identity derive`
  - `identity list`
  - `identity recover`

### Phase 2: P1 签名集成

状态：已完成

- 允许根 HD 身份文件为子作者签名
- 增加 HD 元数据：
  - `hd.parent`
  - `hd.parent_pubkey`
  - `hd.path`
- 在消息来源校验里加入派生公钥验证
- 保持独立身份行为不变

### Phase 3: P1 信任模型

状态：进行中

- 扩展 writer policy，使其理解父子作者关系
- 保持黑名单优先于白名单
- 为可信子身份增加可选的派生验证思路

### Phase 4: P2 API 与 SDK

状态：进行中

- 评估本地 HTTP API 是否应暴露 HD 身份管理
- 在核心行为稳定前，暂缓 Python / JS SDK 的改动

### Phase 5: 文档

状态：待完成

- 说明 HD 身份生命周期与安全备份规则
- 说明助记词和私钥不会在交互输出中回显

## 这一轮范围

- 完成 Phase 1
- 完成 Phase 2 的核心签名与校验部分
- 补齐派生、CLI 和兼容性测试
- 如果 writer policy 扩展风险过高，则延后到下一轮

## 风险

- Ed25519 的 SLIP-0010 只支持 hardened 派生，CLI 和路径解析必须强约束
- hardened Ed25519 无法仅凭父公钥证明子密钥来源
- 只保存公开元数据的子身份文件不能直接签名，除非显式导出带私钥材料的版本
- Author URI 到派生路径的映射必须确定且文档化
