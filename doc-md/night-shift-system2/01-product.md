# 01 产品目标

## 1.1 产品名称

- 中文名：`夜间快讯值班系统2`
- 英文代号：`night-shift-system2`

## 1.2 产品目标

做一个**独立本地运行**的值班系统，服务夜间快讯值守场景。

核心目标：

- 把夜班值守的关键动作收口到一个程序里
- 降低“靠聊天和记忆维持值班”的混乱程度
- 让终审、复核、事故、交接、简报形成统一的可追溯面板
- 在不依赖外部数据库、外部服务的前提下，本地即可运行和验证

## 1.3 用户角色

### 值班主编

负责：

- 决定发不发
- 定口径
- 确认事故恢复是否结束
- 确认交接是否完成

### 夜班编辑

负责：

- 收集来源
- 填来源状态
- 起草简报
- 推进交接项

### 复核员

负责：

- 标记风险
- 补充 review 意见
- 提醒缺失条件

说明：

- 这里的“复核员”可以是人，也可以是 AI agent
- 但程序本身不依赖 Team、Agent 框架

## 1.4 非目标

当前版本不做：

- 多人实时协同编辑
- 权限系统
- 复杂账号体系
- 远程同步
- 外部消息抓取
- 真正自动发稿
- 复杂的排班系统

## 1.5 最小成功标准

当以下条件都成立时，视为第一版完成：

1. 可以本地启动一个 Web 程序
2. 可以录入待核来源
3. 可以把来源推进到：
   - 待核
   - 已核
   - 暂缓
   - 放弃
4. 可以形成终审决策
5. 可以记录 review / risk
6. 可以记录 incident / recovery
7. 可以生成夜班交接内容
8. 可以导出一份夜间简报
9. 程序重启后数据仍然存在

## 1.6 推荐技术落点

默认推荐：

- 单二进制本地服务
- 本地 Web UI
- 文件存储
- JSON 持久化

推荐原因：

- 风险低
- 容易本地运行
- 易调试
- 任意模型都容易稳定实现
