# 夜间快讯值班系统2 产品规格

## 1. 产品定义

`夜间快讯值班系统2` 是一个本地运行的夜班值守工作台，用来处理：

- 快讯来源筛选
- 风险复核
- 终审决策
- 夜间简报沉淀
- 事故处置
- 交接给早班

它不是聊天系统，不是 Team 的替代实现，也不是知识库。

它是一个**围绕夜班工作流**设计的独立本地程序。

## 2. 目标用户

### 主用户

- 夜班编辑
- 值班主编
- 复核员
- 早班接手人

### 默认运行方式

一个人可以在本地扮演多个角色。

程序不要求多人同时在线，也不要求账号系统。

## 3. 非目标

本期不做：

- 即时聊天
- 多用户实时协作
- 权限系统
- 外部推送平台对接
- 自动抓取新闻源
- 大模型自动生成结论

## 4. 核心模块

### 4.1 值班总览

必须展示：

- 当前班次概况
- 待处理来源数
- 待复核数
- 待决策数
- 待交接数
- 事故状态
- 最近动态

### 4.2 来源池

必须支持：

- 新增待处理快讯来源
- 标记来源状态
- 记录来源摘要
- 记录来源可信度
- 记录是否进入终审

### 4.3 复核区

必须支持：

- 对某条来源填写 review
- 填写 risk
- 填写补充说明
- 形成复核结论

### 4.4 决策区

必须支持：

- 形成正式终审口径
- 对来源给出：
  - 先发
  - 暂缓
  - 丢弃
  - 交早班继续跟进

### 4.5 简报区

必须支持：

- 生成夜间简报
- 维护简报状态
- 记录哪些内容已进入简报

### 4.6 事故区

必须支持：

- 记录事故
- 记录处理中 update
- 记录 recovery
- 挂到待办任务

### 4.7 交接区

必须支持：

- 生成交接事项
- 标记 checkpoint
- 标记 accept
- 输出早班接手列表

### 4.8 任务区

必须支持：

- 列出全部任务
- 按状态筛选
- 修改任务状态
- 关联来源 / 决策 / 事故 / 交接

### 4.9 归档区

必须支持：

- 保存夜间简报
- 保存终审结论
- 保存事故总结
- 保存交接摘要

## 5. 默认页面结构

最少要有这些页面：

1. `/`
- 总览页

2. `/sources`
- 来源池

3. `/reviews`
- 复核区

4. `/decisions`
- 决策区

5. `/briefs`
- 简报区

6. `/incidents`
- 事故区

7. `/handoffs`
- 交接区

8. `/tasks`
- 任务页

9. `/archive`
- 归档页

## 6. 默认技术落点

为减少实现分叉，规格默认约束为：

- 本地 Web 应用
- 本地 HTTP 服务
- HTML 页面 + 简单表单/按钮
- 本地 JSON 或 SQLite 持久化

## 7. 最小完成标准

当以下条件全部满足，才能算“夜间快讯值班系统2 已完成”：

- 能启动本地服务
- 能新增至少 3 条来源
- 能完成 1 条复核
- 能形成 1 条终审结论
- 能生成 1 份夜间简报
- 能记录 1 次事故并完成 recovery
- 能生成 1 份交接
- 能在任务区看到自动形成的任务
- 能在归档区看到输出结果
