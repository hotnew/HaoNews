### 已有能力

```

  team/{teamID}/team.json          — 团队元信息

  team/{teamID}/artifacts.jsonl    — 产物列表（JSONL）
  team/{teamID}/history.jsonl      — 变更审计日志（JSONL）
  team/{teamID}/archives/{id}.json — 归档快照

数据模型:
  Info / Member / Policy / Channel / Message / Task / Artifact / ChangeEvent / ArchiveSnapshot

已实现:
  ✅ 文件锁（syscall.Flock via withTeamLock）
  ✅ JSONL 尾部读取优化（readLastJSONLLines）
  ✅ 成员去重与归一化
  ✅ 归档快照（手动触发）
  ✅ 变更历史记录（ChangeEvent）
  ✅ HTTP API + 页面（handler 层完整）
```

### 缺失的能力（对照 A2A 协议）

```
A2A 有、Team 没有:
  ❌ Agent Card（能力声明与自描述）

  ❌ Message 签名验证（Team 消息无签名字段强制执行）
  ❌ Context（多轮对话上下文 ID）
  ❌ Part 类型系统（text / file / data 多态内容）
  ❌ Artifact 与 Task 的绑定生命周期
  ❌ 推送通知（异步任务完成回调）
  ❌ 流式更新（SSE / streaming 实时状态推送）
  ❌ 跨 Team 引用
  ❌ P2P 同步（Team 数据当前纯本地）
```

```
      // 1. 检查分片目录是否存在

      if info, err := os.Stat(shardDir); err == nil && info.IsDir() {
          return s.loadMessagesFromShards(shardDir, limit)

  func (s *Store) loadMessagesFromShards(shardDir string, limit int) ([]Message, error) {
      entries, _ := os.ReadDir(shardDir)
      // 按文件名倒序排列（最新周在前）
      sort.Slice(entries, func(i, j int) bool {
          return entries[i].Name() > entries[j].Name()
      })
      var out []Message
      for _, entry := range entries {
          if !strings.HasSuffix(entry.Name(), ".jsonl") {
              continue
          }
          path := filepath.Join(shardDir, entry.Name())
          lines, err := readLastJSONLLines(path, limit-len(out))
          if err != nil {
              continue
          }
          for _, line := range lines {
              var msg Message
              if json.Unmarshal([]byte(line), &msg) == nil {
                  out = append(out, msg)
              }
          }
          if limit > 0 && len(out) >= limit {
              break
          }
      }
      // 排序
      sort.SliceStable(out, func(i, j int) bool {
          return out[i].CreatedAt.After(out[j].CreatedAt)
      })
      if limit > 0 && len(out) > limit {
          out = out[:limit]
      }
      return out, nil
  }
```

### 1.3 Handler 并行加载

**问题**：`handleTeam()` 串行执行 9 次文件 I/O。

**实施步骤**：

```
步骤 1: 打开 internal/plugins/haonewsteam/handler.go

步骤 2: 创建通用并行加载工具:

  // 在 handler.go 顶部或单独 util.go 中
  type loadResult[T any] struct {
      value T
      err   error
  }

  func loadParallel[T any](ctx context.Context, fns ...func() (T, error)) ([]T, error) {
      results := make([]loadResult[T], len(fns))
      var wg sync.WaitGroup
      wg.Add(len(fns))
      for i, fn := range fns {
          go func(idx int, f func() (T, error)) {
              defer wg.Done()
              v, err := f()
              results[idx] = loadResult[T]{value: v, err: err}
          }(i, fn)
      }
      wg.Wait()
      out := make([]T, len(results))
      for i, r := range results {
          if r.err != nil {
              return nil, r.err
          }
          out[i] = r.value
      }
      return out, nil
  }
```

```
  func verifyMessageSignature(msg Message) bool {
      if msg.OriginPublicKey == "" || msg.Signature == "" {
          return false
      }
      pubKeyBytes, err := hex.DecodeString(msg.OriginPublicKey)
      if err != nil || len(pubKeyBytes) != ed25519.PublicKeySize {
          return false
      }
      sigBytes, err := hex.DecodeString(msg.Signature)
      if err != nil {
          return false
      }
      // 签名覆盖范围：author + channel + content + created_at
      payload := fmt.Sprintf("%s|%s|%s|%s",
          msg.AuthorAgentID, msg.ChannelID, msg.Content,
          msg.CreatedAt.UTC().Format(time.RFC3339))
      return ed25519.Verify(pubKeyBytes, []byte(payload), sigBytes)
  }
```

### 2.2 Task 状态机（对齐 A2A TaskState）

**目标**：引入明确的任务状态转换规则，防止非法状态变更。

**A2A 定义的 7 个状态**：
```
submitted → working → completed (终态)
                   → failed    (终态)
                   → canceled  (终态)
         → input_required (中断态，可恢复)
         → auth_required  (中断态，可恢复)
         → rejected       (终态)
```

**实施步骤**：

```
步骤 1: 定义状态常量和转换规则:

  // internal/haonews/team/task_state.go （新文件）
  package team

  const (
      TaskStateOpen         = "open"          // 对应 A2A submitted
      TaskStateDoing        = "doing"         // 对应 A2A working
      TaskStateBlocked      = "blocked"       // 对应 A2A input_required
      TaskStateDone         = "done"          // 对应 A2A completed
      TaskStateFailed       = "failed"        // 对应 A2A failed
      TaskStateCancelled    = "cancelled"     // 对应 A2A canceled
      TaskStateRejected     = "rejected"      // 对应 A2A rejected
  )

  // 默认状态转换表
  var defaultTaskTransitions = map[string][]string{
      TaskStateOpen:      {TaskStateDoing, TaskStateCancelled, TaskStateRejected},
      TaskStateDoing:     {TaskStateDone, TaskStateFailed, TaskStateBlocked, TaskStateCancelled},
      TaskStateBlocked:   {TaskStateDoing, TaskStateCancelled},
      TaskStateDone:      {TaskStateOpen},  // 允许 reopen
      TaskStateFailed:    {TaskStateOpen},  // 允许 retry
      TaskStateCancelled: {TaskStateOpen},  // 允许 reopen
      TaskStateRejected:  {},               // 终态，不可变
  }

  func IsTerminalState(state string) bool {
      switch state {
      case TaskStateDone, TaskStateFailed, TaskStateCancelled, TaskStateRejected:
          return true
      }
      return false
  }

  func IsValidTransition(from, to string) bool {
      allowed, ok := defaultTaskTransitions[from]
      if !ok {
          return false
      }
      for _, s := range allowed {
          if s == to {
              return true
          }
      }
      return false
  }

步骤 2: 在 Policy 中支持自定义转换:

  type Policy struct {
      // ... 现有字段 ...
      TaskTransitions map[string]TaskTransitionRule `json:"task_transitions,omitempty"` // ← 新增
  }

  type TaskTransitionRule struct {
      AllowedNext  []string `json:"allowed_next"`
      AllowedRoles []string `json:"allowed_roles,omitempty"`
  }

步骤 3: 在 SaveTask 中强制检查状态转换:

  func (s *Store) SaveTask(teamID string, task Task) error {

              }
          }
          // ... 现有保存逻辑 ...
      })
  }

步骤 4: 添加 Task 状态变更历史:
```

### 3.2 实时事件推送（SSE）

**目标**：Agent 可以订阅 Team 的实时变更，不需要轮询。

**实施步骤**：

```
步骤 1: 添加 SSE 端点:

  // internal/plugins/haonewsteam/handler.go
  func handleTeamSSE(store *teamcore.Store, teamID string, w http.ResponseWriter, r *http.Request) {
      flusher, ok := w.(http.Flusher)
      if !ok {
          http.Error(w, "streaming not supported", http.StatusInternalServerError)
          return
      }

      w.Header().Set("Content-Type", "text/event-stream")
      w.Header().Set("Cache-Control", "no-cache")
      w.Header().Set("Connection", "keep-alive")

      // 订阅 team 事件
      ch := store.Subscribe(teamID)
      defer store.Unsubscribe(teamID, ch)

      for {
          select {
          case event := <-ch:
              data, _ := json.Marshal(event)
              fmt.Fprintf(w, "event: %s\ndata: %s\n\n", event.Type, data)
              flusher.Flush()
          case <-r.Context().Done():
              return
          }
      }
  }

步骤 2: 在 Store 中添加事件总线:

  type Store struct {
      root        string
      subscribers map[string][]chan TeamEvent  // teamID → channels
      subMu       sync.RWMutex
  }

  type TeamEvent struct {
      Type      string `json:"type"`   // "message" | "task_update" | "member_change" | "artifact"
      TeamID    string `json:"team_id"`
      ChannelID string `json:"channel_id,omitempty"`
      SubjectID string `json:"subject_id,omitempty"`
      Summary   string `json:"summary,omitempty"`
      Timestamp time.Time `json:"timestamp"`
  }

  func (s *Store) Subscribe(teamID string) chan TeamEvent {
      s.subMu.Lock()
      defer s.subMu.Unlock()
      ch := make(chan TeamEvent, 32)
      s.subscribers[teamID] = append(s.subscribers[teamID], ch)
      return ch
  }

  func (s *Store) Unsubscribe(teamID string, ch chan TeamEvent) { ... }

  func (s *Store) Publish(event TeamEvent) {
      s.subMu.RLock()
      defer s.subMu.RUnlock()
      for _, ch := range s.subscribers[event.TeamID] {
          select {
          case ch <- event:
          default: // 丢弃慢消费者
          }
      }
  }

步骤 3: 在所有写操作后触发 Publish:
  - AppendMessage 后: Publish(TeamEvent{Type: "message", ...})
  - SaveTask 后: Publish(TeamEvent{Type: "task_update", ...})
  - SaveMembers 后: Publish(TeamEvent{Type: "member_change", ...})

步骤 4: 路由注册:
  GET /api/teams/{teamID}/events → handleTeamSSE

步骤 5: 运行 go test -race
```

## 九、设计原则总结

1. **文件即数据库**：保持 JSONL / JSON 作为权威存储，Redis 仅做缓存，保证任何人/Agent 可直接读文件理解状态
2. **签名即信任**：所有写操作可选签名验证，Team 历史本身就是可验证的审计链
3. **追加不可变**：消息和历史只追加不修改，任务通过追加新版本实现更新，旧版本保留
4. **渐进兼容**：所有新字段都是 `omitempty`，旧数据无需迁移即可读取；新功能通过 Policy 开关控制
5. **模块边界**：Team 不依赖 Live/Topics，跨模块只走 Reference 引用，不走数据耦合
6. **A2A 对齐但不照搬**：借鉴 A2A 的 Task 状态机、Agent Card、Part 类型系统、Context 概念，但保持 P2P + 文件优先的本地特色
