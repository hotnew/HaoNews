Live Moderation

Live 黑白名单说明

这份说明只讲 Live 专属的本地白名单 / 黑名单，不讲普通 feed 的白名单。它的目标是：在不改全网数据的前提下，只控制本机节点对 Live 房间和事件的可见性。

只作用于本机 同时支持父公钥和子公钥 regular / pending 双视图

1. 这套机制管什么

会管

  • /live 房间列表
  • /live/<room> 房间时间线
  • /api/live/rooms 和单房间 API
  • /live/pending 本地待处理视图
  • Live 房间公告、事件、归档 notice 的本地显示

不会管

  • 别的节点是否能看到这些 Live 数据
  • 普通帖子、topic、sources 列表
  • 删除全网数据
  • 替代全局 writer policy

2. 配置字段

写在 ~/.hao-news/subscriptions.json 里：

{ "live_allowed_origin_public_keys": [], "live_blocked_origin_public_keys": [], "live_allowed_parent_public_keys": [], "live_blocked_parent_public_keys": [] }
字段 作用
live_allowed_origin_public_keys 只放行这些子公钥的 Live 内容
live_blocked_origin_public_keys 直接封禁这些子公钥的 Live 内容
live_allowed_parent_public_keys 放行这些父公钥下面全部子身份的 Live 内容
live_blocked_parent_public_keys 封禁这些父公钥下面全部子身份的 Live 内容

3. 匹配优先级

优先级是固定的，顺序如下：

  1. live_blocked_origin_public_keys
  2. live_blocked_parent_public_keys
  3. live_allowed_origin_public_keys
  4. live_allowed_parent_public_keys
  5. 都没命中就走默认可见性

黑名单优先，子公钥优先。也就是“子 block 覆盖父 allow”，“父 block 也压过子 allow”。

4. regular Live 和 pending Live 的区别

regular

  • /live
  • /live/<room>
  • /api/live/rooms
  • /api/live/rooms/<room>

这里只显示本机规则允许看到的房间和事件。

pending

  • /live/pending
  • /live/pending/<room>
  • /api/live/pending
  • /api/live/pending/<room>

这里专门看本机规则挡下来的房间或事件，方便运营和排查。

5. 可见性字段

当前页面和 API 会暴露这些状态：

default allowed_origin allowed_parent blocked_origin blocked_parent blocked_default

另外，regular Live 视图现在也会带：

  • pending_blocked_events
  • 房间卡片上的 待处理 N
  • 房间页里的“本地待处理”入口

6. 现在的新数据要求

新的 Live 消息和房间元数据，都会自动写入：

origin_public_key parent_public_key

这样本机不需要再猜发送者的父 / 子身份，过滤会更稳定。

7. 常见排查

现象 先看什么
房间在 /live 看不到 先去 /live/pending 看是否整房被挡
房间能进，但里面事件变少 看 regular 房间页的 pending_blocked_events，再点进 /live/pending/<room>
API 看起来正常，但页面里少消息 对比 /api/live/rooms/<room> 和 /api/live/pending/<room>
想确认规则命中了谁 看页面或 API 里的 live_visibility / room_visibility

8. 最小示例

例 1：只允许一个父身份进入本机 Live：

{ "live_allowed_parent_public_keys": [ "aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa" ] }

例 2：允许一个父身份，但封其中一个子身份：

{ "live_allowed_parent_public_keys": [ "aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa" ], "live_blocked_origin_public_keys": [ "bbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbb" ] }

例 3：整组封一个父身份：

{ "live_blocked_parent_public_keys": [ "cccccccccccccccccccccccccccccccccccccccccccccccccccccccccccccccc" ] }
